他のプロトコルを用いて Gfarm ファイルシステムにアクセスする

English | 日本語

1. 概要

ファイル共有／公開サーバアプリケーションを経由して，Gfarm ファイルシステムをエクスポートできるかどうか，いくつか調査しました． 各サーバに対して，gfarm2fs を組み合わせれば実現できます．

2. 組み合わせ調査結果

Servers syscall-hooking library GfarmFS-FUSE
Samba ◎ ◎
allow_root 必要
NFS × ○
allow_root 必要
vsftpd ○ ○
ProFTPD ○ ○
allow_root 必要
globus-gridftp-server
(GT4) (root 権限起動）		 ○
サーバ側でも grid-proxy-init 必要		 ○
サーバ側でも grid-proxy-init 必要
Apache HTTP Server
(Gfarm にアクセスできる
ユーザの権限で起動)		 ○
httpd.conf に
EnableMMAP Off
EnableSendfile Off
が必要である		 ○

リンク先があれば詳細な解説があります．他のサーバにも応用してみてください．

  • ◎: テスト済み
  • ○: 簡単な動作確認のみ
  • ×: 不可能
  • allow_root 必要とは、FUSE のオプションである -o allow_root を付けてマウントする必要があります．詳しくは FUSE パッケージの README をご覧ください．

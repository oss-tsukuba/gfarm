gfpcopy

Name

gfpcopy -- copy Gfarm files in parallel

Synopsis

gfpcopy [-nqvdpPU] [-f] [-S source-domainname] [-h source-hostfile] [-D destination-domainname] [-H destination-hostfile] [-j num-of-processes] [-w way-of-scheduling] [-W kilobytes-for-threshold] [-s KB/s-to-simulate] [-F num-for-readahead] [-b bufsize] source-url destination-url

DESCRIPTION

gfpcopy copies files and directries specified by the source-url(gfarm:///... or file:///...) to a directory specified by the destination-url(gfarm:///... or file:///...) in parallel.

When a set of source hosts are specified, only files stored on the set of source hosts are copied. A set of hosts can be specified by a domain name and/or a host file. When both a domain name and a host file are specified, a set of hosts is determined by both conditions.

If the source-url parameter is a directory, files under the directory will be copied recursively.

gfpcopy can also read the directory and get informations of the entries in parallel.

GFPCOPY OPTIONS

These are the options only for gfpcopy.

-b bufsize

Specifies the buffer size to copy.

The default value is 64KiB (64 * 1024).

-f

If a file or directory exists on the destination, the file or directory is overwritten.

COMMON OPTIONS

gfpcopy has the same options with gfprep. (see man gfprep)

EXAMPLES

To copy files under the directory recursively.

$ gfpcopy gfarm:///dir file:///tmp/dir
$ gfpcopy file:///tmp/dir gfarm:///dir
$ gfpcopy gfarm:///dir1 gfarm:///dir2

To copy a file.

$ gfpcopy gfarm:///dir/file file:///dir

NOTES

If the gfmd and gfpcopy are far apart, the gfpcopy operation is slow. Specifying a large value by the -j option or client_parallel_copy in gfarm2.conf may be effective in such a case.
